TLT Yield-Driven Strategies

 

Trading symbol: TLT    

Trading signal: 10-Year U.S. Treasury Yield (10Y)

 

Strategy Overview:

The relationship between TLT and the 10-Year Treasury Yield is inverse:

If 10Y yield rises → expect TLT to drop → short/sell TLT.

If 10Y yield falls → expect TLT to rise → buy TLT.

I created 5 strategies in total in a logical order.

 

Strategy 1:  Short and Long Moving Average (with 1-Day Lag)

Code is in Strategy1_TLT_and_10y_bond.py

Idea:

Use short-term and long-term moving averages of the 10Y yield to generate trading signals for TLT.

 

Rules:

If the short-term MA > long-term MA → yield is trending upward → short TLT.

If the short-term MA < long-term MA → yield is trending downward → buy TLT.

 


 

This strategy applies a 1-day lag to avoid look-ahead bias, meaning signals are generated after market close and executed at the next day’s open.

 

 

 

Strategy 2: Momentum and Z-Score (with 1-Day Lag):

Code is in Strategy2_TLT_and_10y_bond_momentum_and_Z_score .py

 

Idea:

Use recent yield momentum and Z-score to capture directional changes.

 

Rules:

Momentum: If yield increases over the last few days → short TLT; if yield decreases → buy TLT.

Z-score: If yield is significantly above its recent mean → short TLT; if below → buy TLT.

 

 

 

 

 

 

The momentum and Z-score strategy performed worse than the moving average strategy, with a lower Sharpe ratio and less stability.

 

 

We can see that the sharpe ratio of the first strategy is far more better than the momentum strategy and the Z-score strategy. So I chose the short long MA strategy as my main strategy and I want to improve the strategy to make it more profitable. The first thing I want to try is to backtest this strategy using historical data without any lag and try to look-ahead today’s data, which was implemented in Strategy 3 as below.

 

 

 

Strategy 3: Short and Long Moving Average (without Lag):

Code is in Strategy3_TLT_and_10y_bond_MA.py

 

Idea:
Test the moving average strategy without any lag—using the current day’s yield to simulate having access to today’s information in advance.

 

 

 

 

 

 

 

Strategy 3 performs far more better than Strategy 1, with higher sharpe ratio and annual return. In this way, we can see that if we can know today’s yield ahead, then we can definitely improve the strategy’s performance. However, this creates look-ahead bias and is not realistic in live trading. So we need to accurately predict next day’s yield. I came up with the idea to predict the yield at day t+1 based on the historical data up to day t using LSTM. But in this strategy I trained the LSTM model only once on the training data set and use this model to predict on the test set, which is exactly what Strategy 4 is doing.

 

 

 

Strategy 4:  LSTM Prediction of Next Day’s Yield

Code is in Strategy4_TLT_and_10y_bond_LSTM.py (Here I defined all the model training and evaluation in utils.py )

 

Idea:
Use an LSTM model to predict the next day’s yield (t+1) using historical data up to day t.

 

Implementation:

Train the LSTM model once on the training dataset.

Use the trained model to predict the entire test set.

 

 

 

The LSTM model was not accurate enough to capture daily yield changes, so trading performance was limited. The static model could not adapt well to changing market conditions. So I am thinking of implementing a rolling forecast approach — continuously retraining the model on the most recent data and generating updated predictions at each step—to adapt to changing market conditions and reduce lag between yield movements and TLT price reactions. So here comes Strategy 5.

 

 

Strategy 5: LSTM Rolling Forecast (Adaptive Model)

Code is in Strategy5_LSTM_rolling_forecast_backtest.py (Here I defined all the model training and evaluation in utils.py )

 

 

Idea: 

Continuously retrain the LSTM model on the most recent data to predict the next day’s yield. This rolling forecast adapts to new market information and regime changes.

 

 

 

Process:

1. At each step, train (or update) the LSTM model using the most recent window of yield data.

2. Predict the next day’s yield.

3. If predicted yield rises → short TLT; if predicted yield falls → buy TLT.

4. Execute the trade automatically at the next market open.

 

 

 

We can run Strategy5_LSTM_rolling_forecast_backtest.py to see the result of the model training and also the backtesting trading performance, allowing us to evaluate both the predictive accuracy and the profitability of the rolling forecast strategy.
